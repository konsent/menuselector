# 메뉴 카테고리 기본 닫힘 설계

## 목표

서비스를 처음 열었을 때 메뉴 목록의 모든 카테고리를 닫힌 상태로 표시한다. 현재 첫 카테고리인 `찌개`가 자동으로 열리는 동작을 제거한다.

## 범위

- 메뉴 목록의 초기 렌더링 상태만 변경한다.
- 사용자가 카테고리 제목을 눌러 여닫는 동작은 유지한다.
- 검색어 입력 시 일치하는 메뉴가 있는 카테고리를 자동으로 여는 현재 동작은 유지한다.
- 재료 우선 모드의 카테고리 동작과 저장 데이터는 변경하지 않는다.

## 구현 설계

`app.js`의 메뉴 카테고리 생성 로직에서 첫 번째 카테고리에 부여하는 초기 확장 상태를 제거한다. 모든 메뉴 카테고리 제목은 초기값으로 `aria-expanded="false"`를 가지며, 어느 섹션에도 `active` 클래스를 추가하지 않는다.

카테고리 클릭과 키보드 조작은 기존처럼 `active` 클래스를 토글하고, 이에 맞춰 `aria-expanded` 값을 갱신한다. 검색 필터링 로직은 검색어가 있고 결과가 존재하는 섹션에 `active` 클래스를 부여하므로 그대로 둔다.

## 검증

1. 새로고침 직후 `찌개`를 포함한 모든 메뉴 카테고리의 항목이 숨겨져 있는지 확인한다.
2. 카테고리 제목 클릭 및 Enter 키로 각 카테고리가 열리고 닫히는지 확인한다.
3. 메뉴 검색 시 결과가 있는 카테고리가 열리고, 검색어를 지우면 기존 필터 동작으로 돌아가는지 확인한다.

## 결정

초기 상태를 모든 카테고리 닫힘으로 통일한다. 마지막으로 연 카테고리를 저장·복원하는 기능은 요구 범위에 포함하지 않는다.
